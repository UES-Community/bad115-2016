El Lenguaje de Consulta Estructurado (SQL) opera bajo un paradigma declarativo: el desarrollador especifica el **"qué"** (el resultado deseado) y delega al optimizador del DBMS el **"cómo"** (el plan de ejecución físico).

Para dominar SQL, es crucial distinguir entre tres cronologías:
1. **Orden Sintáctico:** La secuencia gramatical requerida al escribir la consulta (inicia con `SELECT`).
2. **Orden Lógico:** El orden en el que el motor de base de datos procesa conceptualmente la consulta (dictado por el estándar ANSI SQL).
3. **Orden Físico:** La secuencia real de ejecución decidida por el optimizador (usando índices, estadísticas, etc.) para minimizar el I/O.

---

### Orden Lógico de Procesamiento

El orden lógico define cómo se analizan los datos y cómo las tablas virtuales transitorias (VT) se transfieren de una fase a la siguiente. A continuación se detallan las fases en su secuencia lógica real:

1. **FROM / ON / JOIN:** Delimita el espacio de trabajo. Se calculan los productos cartesianos y se aplican las condiciones `ON`, generando la primera tabla virtual.
2. **WHERE:** Aplica un filtro por fila sobre la tabla virtual previa. Las tuplas cuya condición sea falsa o devuelva `NULL` (lógica ternaria) son descartadas.
3. **GROUP BY:** Agrupa las filas según las columnas declaradas. A partir de aquí, las columnas que no formen parte de la clave de agrupación deben encapsularse obligatoriamente en funciones de agregación (como `SUM`, `AVG`, `COUNT`).
4. **WITH CUBE / ROLLUP:** Genera subtotales analíticos y supergrupos de resumen.
5. **HAVING:** Filtro exclusivo para los grupos generados en la fase 3 (ej. `HAVING SUM(ventas) > 50`).
6. **SELECT:** Se ejecuta la proyección, seleccionando las columnas deseadas y evaluando las expresiones matemáticas. Aquí es donde se crean y reconocen los alias de columnas.
7. **Window Functions (Funciones de Ventana):** Se evalúan las funciones analíticas con `OVER()` (como `ROW_NUMBER()` o `RANK()`) sobre el resultado del `SELECT`.
8. **DISTINCT:** Elimina filas duplicadas para cumplir con la teoría de conjuntos.
9. **ORDER BY:** Clasifica el conjunto final de datos (ascendente o descendente). Es la única cláusula que puede referenciar alias definidos en el `SELECT` debido a su orden sucesivo.
10. **OFFSET / FETCH / TOP / LIMIT:** Trunca el conjunto de resultados, devolviendo únicamente la cantidad de registros solicitados (ideal para paginación).

### Implicaciones Prácticas

Comprender esta secuencia elimina errores comunes. Por ejemplo:
* **¿Por qué no se puede usar un alias del `SELECT` en el `WHERE`?** Porque el `WHERE` (Fase 2) se procesa mucho antes que el `SELECT` (Fase 6). El motor aún no conoce el alias.
* **¿Cómo filtrar por campos calculados dinámicamente?** Se debe encapsular la lógica en una Expresión de Tabla Común (CTE) o en una subconsulta para forjar una tabla intermedia que sí pueda ser evaluada en el ciclo externo.
