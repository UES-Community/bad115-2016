La estructura jerárquica y anidada del XML difiere del modelo plano de bases de datos relacionales. Para interactuar eficientemente con documentos XML, el W3C desarrolló un ecosistema de tecnologías sustentado en el modelo de datos común **XDM (XQuery and XPath Data Model)**, que conceptualiza los documentos como árboles jerárquicos de nodos tipados.

---

### XPath (XML Path Language)

XPath es el lenguaje de expresiones utilizado para navegar por la estructura de árbol de un documento XML y seleccionar subconjuntos de nodos. Es un lenguaje de consulta puro (no modifica los datos) y sirve de base para XQuery y XSLT.

#### Estructura de un "Paso de Ubicación" (Location Step)
Cada paso de navegación en XPath consta de tres componentes principales:
1. **Eje (*Axis*):** Define la dirección de la navegación en el árbol a partir del nodo actual (ej. hijos, descendientes, ancestros).
2. **Prueba de Nodo (*Node Test*):** Filtra los nodos localizados por el eje según su tipo o nombre (ej. `node()`, `text()`, `*`, o etiquetas específicas como `libro`).
3. **Predicados (*Predicates*):** Condiciones lógicas escritas entre corchetes `[...]` que filtran los nodos seleccionados. Aceptan operadores lógicos, matemáticos y funciones.

```xpath
-- Ejemplo conceptual de expresión XPath
/bookstore/book[price < 30 and author='Smith']/title
```

#### Ejes Principales de XPath y sus Abreviaciones

| Eje XPath (*Axis*) | Dirección y Semántica de Navegación | Sintaxis Abreviada |
| :--- | :--- | :--- |
| `child::` | Selecciona los nodos hijo directos del nodo de contexto. | `/` (o se omite) |
| `descendant-or-self::` | Selecciona el nodo actual y todos sus descendientes en profundidad. | `//` |
| `attribute::` | Selecciona los atributos del elemento actual. | `@` |
| `parent::` | Selecciona el padre inmediato del nodo actual. | `..` |
| `self::` | Selecciona el nodo actual. | `.` |
| `following-sibling::` | Selecciona los nodos del mismo nivel (hermanos) posteriores al actual. | *No tiene* |
| `ancestor::` | Selecciona la línea completa de ancestros del nodo (padre, abuelo, raíz). | *No tiene* |

#### Evolución del Estándar
* **XPath 1.0:** Operaba con "conjuntos de nodos" sin tipado estricto, dependiendo de conversiones implícitas de tipos.
* **XPath 2.0 / 3.0 / 3.1:** Incorporó el sistema de tipos de XSD (*Schema-Awareness*), soporte para secuencias homogéneas y heterogéneas, iteradores (`for`), expresiones condicionales (`if-then-else`), aserciones cuantificadas (`some`/`every`), mapeo JSON y soporte avanzado de funciones.

### XQuery (XML Query Language)

XQuery es a los datos semiestructurados en XML lo que SQL es a las tablas relacionales. Es un superconjunto de XPath 2.0 diseñado para extraer información de bases de datos XML (tanto documentos individuales como colecciones complejas).

#### Expresiones FLWOR
El núcleo operativo de XQuery es la estructura **FLWOR**, que permite realizar proyecciones, ordenamientos y combinaciones de datos (*joins*):

* **`for`:** Inicia ciclos iterativos que asocian una variable a cada nodo devuelto por una expresión XPath (similar a un bucle `foreach`).
* **`let`:** Vincula una variable a una secuencia completa como un bloque único, evitando evaluaciones repetitivas dentro de ciclos.
* **`where`:** Aplica filtros booleanos sobre las variables definidas en `for` o `let` (análogo al `WHERE` de SQL).
* **`order by`:** Ordena secuencialmente los elementos resultantes antes de retornarlos.
* **`return`:** Define la salida que se va a construir en cada iteración, permitiendo formatear XML dinámicamente.

```xquery
(: Ejemplo de consulta FLWOR en XQuery :)
for $b in doc("libros.xml")/bookstore/book
where $b/price < 30
order by $b/title
return
  <libro_barato isbn="{$b/@isbn}">
    {$b/title}
  </libro_barato>
```

### XSLT (Extensible Stylesheet Language Transformations)

A diferencia de XQuery (orientado a consultas), XSLT está diseñado para la transformación estructural de documentos XML hacia otros formatos como HTML, texto plano (CSV), JSON o documentos PDF (a través de XSL-FO).

#### Paradigma Basado en Plantillas
XSLT es un lenguaje declarativo guiado por el documento (*document-driven*). El procesador recorre el árbol XML y aplica reglas específicas cuando un nodo coincide con un patrón definido en una plantilla:

* **`<xsl:template match="...">`:** Define la regla de transformación aplicable a los nodos que coincidan con la expresión XPath.
* **`<xsl:apply-templates>`:** Delega el procesamiento de los nodos hijo a otras plantillas del documento.

```xml
<!-- Ejemplo simple de hoja de estilos XSLT -->
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:template match="/">
    <html>
      <body>
        <h2>Lista de Libros</h2>
        <xsl:apply-templates select="bookstore/book"/>
      </body>
    </html>
  </xsl:template>

  <xsl:template match="book">
    <p><xsl:value-of select="title"/> - <xsl:value-of select="price"/></p>
  </xsl:template>
</xsl:stylesheet>
```

#### Agrupación en XSLT 2.0 / 3.0 (`<xsl:for-each-group>`)
La agrupación estructural compleja (que en XSLT 1.0 requería el complejo método *Muenchian* usando funciones `generate-id()`) se simplificó sustancialmente mediante las siguientes directivas:

* **`group-by`:** Agrupa elementos según el valor único de una clave de agrupación común.
* **`group-adjacent`:** Agrupa elementos contiguos que comparten el mismo valor de clave (ideal para agrupar párrafos u elementos correlativos).
* **`group-starting-with` / `group-ending-with`:** Agrupa secuencias de nodos basándose en un nodo delimitador inicial o final (como cabeceras `<h2>`).
* **Funciones de apoyo:** `current-group()` devuelve los nodos agrupados en la iteración actual y `current-grouping-key()` extrae el valor del criterio que generó el grupo.

### Comparativa: XQuery vs. XSLT

| Característica | XQuery | XSLT |
| :--- | :--- | :--- |
| **Enfoque Principal** | Búsqueda y extracción de datos (*Query-driven*). | Transformación de formato y estructura (*Document-driven*). |
| **Sintaxis** | Compacta y legible (similar a SQL y lenguajes funcionales). | XML Verboso (todas las etiquetas del script son XML válido). |
| **Bifurcaciones Lógicas** | Estructuras directas: `if (cond) then A else B`. | Bloques estructurados XML: `<xsl:choose>`, `<xsl:when>`, `<xsl:otherwise>`. |
| **Asignación de Variables** | Sintaxis ligera: `let $x := valor`. | Elemento XML: `<xsl:variable name="x" select="valor"/>`. |
| **Caso de Uso Común** | Consultas a bases de datos nativas XML (NXD) y orígenes heterogéneos. | Generación de interfaces web (HTML), reportes (PDF) y mapeo entre esquemas. |
