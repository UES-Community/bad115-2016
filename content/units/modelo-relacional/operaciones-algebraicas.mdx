El arsenal operativo del álgebra relacional se clasifica según el número de relaciones de entrada (unarios o binarios) y su origen epistemológico (primitivos de la teoría de conjuntos o específicos del modelo relacional).

Las operaciones fundamentales determinan la capacidad computacional de la base de datos, mientras que las operaciones derivadas proporcionan atajos semánticos.

| Grupo de Operación | Operador y Símbolo | Naturaleza y Mecanismo Funcional | Dependencia Semántica |
| :--- | :--- | :--- | :--- |
| **Operadores Unarios** | Selección / Restricción ($\sigma$) | Partición horizontal de una relación según un predicado lógico de filtrado. | Primitivo, específico relacional |
| | Proyección ($\pi$) | Partición vertical extrayendo columnas específicas y eliminando duplicados. | Primitivo, específico relacional |
| | Renombramiento ($\rho$) | Renombra la relación resultante o sus columnas para evitar colisiones. | Primitivo, específico relacional |
| **Operadores Binarios Conjuntistas** | Unión ($\cup$) | Consolida tuplas presentes en ambas relaciones, colapsando redundancias. | Primitivo, Teoría de Conjuntos |
| | Intersección ($\cap$) | Extrae las tuplas presentes simultáneamente en ambas relaciones. | Derivado, Teoría de Conjuntos |
| | Diferencia / Resta ($-$) | Sustrae las tuplas del segundo operando de las de la primera relación. | Primitivo, Teoría de Conjuntos |
| | Producto Cartesiano ($\times$) | Multiplica cruzadamente todas las tuplas para forjar todas las combinaciones posibles. | Primitivo, Teoría de Conjuntos |
| **Operadores Binarios Específicos** | Reunión o Join ($\bowtie$) | Concatena atributos evaluando conectores condicionales y preposiciones lógicas. | Derivado relacional |
| | División Relacional ($\div$) | Busca asociaciones exhaustivas bajo restricciones de cuantificadores universales. | Derivado relacional |

### Operadores Unarios

Se aplican sobre una única relación. Son las herramientas de refinamiento más recurrentes, útiles para reducir el volumen de datos antes de realizar operaciones costosas.

* **Selección ($\sigma$):** Funciona como un filtro horizontal. Se define como:
  ```latex
  \sigma[condicion](R)
  ```
  La condición es un predicado booleano que puede contener atributos, constantes y operadores de comparación ($=, \neq, &lt;, &gt;, \le, \ge$) unidos por nexos lógicos ($\land, \lor, \neg$).
  * *Ejemplo:* Filtrar empleados del departamento de ventas: `\sigma[depto = 'Ventas'](Empleados)`.
* **Proyección ($\pi$):** Realiza un filtrado vertical. Se define como:
  ```latex
  \pi[A_1, A_2, \dots, A_n](R)
  ```
  Extrae únicamente los atributos indicados. Debido a la teoría de conjuntos, si al eliminar columnas identificadoras (como llaves primarias) surgen tuplas duplicadas, estas se reducen automáticamente a una sola instancia.
  * *Ejemplo:* Listar solo nombres y salarios: `\pi[nombre, salario](Empleados)`.
* **Renombramiento ($\rho$):** Permite cambiar el nombre de una relación resultante o de sus atributos. Se define como:
  ```latex
  \rho[S(B_1, B_2, \dots, B_n)](R)
  ```
  Es crucial para evitar colisiones de nombres, especialmente al realizar operaciones cruzadas de una relación consigo misma (self-join).

### Operadores Binarios Conjuntistas

Provienen de la lógica de conjuntos. Para que las operaciones de **Unión**, **Intersección** y **Diferencia** sean válidas, las relaciones deben ser **unión-compatibles**:
1. Tener exactamente el mismo grado (mismo número de columnas).
2. Los dominios de las columnas correspondientes deben ser compatibles de izquierda a derecha.

* **Unión ($\cup$):** Adosa ambos conjuntos de tuplas.
  $$R \cup S$$
  Produce una relación con todas las tuplas que están en $R$, en $S$, o en ambas, eliminando duplicados.
* **Intersección ($\cap$):** Conserva solo las tuplas presentes de manera simultánea en ambas relaciones.
  $$R \cap S$$
  Es un operador derivado; se puede definir usando la diferencia: $R \cap S = R - (R - S)$.
* **Diferencia ($-$):** Sustractor asimétrico.
  $$R - S$$
  Produce una relación con todas las tuplas que están en $R$ pero no en $S$. Es sensible al orden de los operandos (no conmutativo).
* **Producto Cartesiano ($\times$):** Combina todas las tuplas del primer operando con todas las tuplas del segundo. No requiere compatibilidad de unión.
  $$R \times S$$
  * *Grado resultante:* Sumativo ($Grade(R) + Grade(S)$).
  * *Cardinalidad resultante:* Multiplicativa ($Card(R) \times Card(S)$).
  * Generalmente es un paso intermedio que requiere un filtrado posterior para adquirir sentido práctico.

### Operadores Derivados Específicos

#### 1. Reunión (Join, $\bowtie$)
Orquestado para reensamblar información normalizada distribuida en múltiples relaciones.

* **Theta-Join ($R \bowtie_\theta S$):** Equivale a un producto cartesiano seguido de una selección según el predicado $\theta$:
  ```latex
  R \bowtie_\theta S = \sigma_\theta(R \times S)
  ```
* **Equi-Join:** Variante del Theta-Join donde el operador de comparación del predicado $\theta$ es la igualdad ($=$).
* **Join Natural ($R \bowtie S$):** Autómata que evalúa automáticamente la igualdad en los atributos con nombres idénticos entre ambas relaciones, eliminando las columnas duplicadas de forma implícita.
* **Outer Joins (Left, Right, Full):** Evitan la pérdida de información de tuplas que no tienen correspondencia en la otra relación, rellenando las columnas faltantes con marcadores nulos (`NULL`).

#### 2. División Relacional ($\div$)
Diseñado para resolver consultas complejas que involucran cuantificación universal ("para todo", $\forall$).

* *Ejemplo:* "Obtener los alumnos que han aprobado todos los cursos obligatorios".
* Si tenemos una relación dividendo $R(X, Y)$ y un divisor $S(Y)$, la división produce $R \div S$ con esquema $X$ tal que para cada valor de $X$ en el resultado, existe una tupla $(x, y)$ en $R$ para cada $y$ en $S$.

##### Simulación algorítmica con operadores primitivos:
Para calcular $R(X, Y) \div S(Y)$:
1. **Obtener candidatos:** Extraer todos los valores posibles de la parte meta $X$:
   $$T_1 = \pi_X(R)$$
2. **Construir el escenario ideal:** Cruzar todos los candidatos con el divisor para crear todas las combinaciones teóricamente requeridas:
   $$T_2 = \pi_X(R) \times S$$
3. **Detectar faltantes:** Encontrar qué combinaciones ideales no existen realmente en nuestra relación original:
   $$T_3 = T_2 - R$$
4. **Filtrar transgresores:** Restar del total de candidatos a aquellos que demostraron tener combinaciones faltantes:
   ```latex
   Resultado = \pi_X(R) - \pi_X(T_3)
   ```
